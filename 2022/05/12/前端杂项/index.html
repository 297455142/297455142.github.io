三、浏览器缓存机制

浏览器缓存是浏览器将用户请求过的静态资源（html、css、js）存储到电脑本地磁盘中，当浏览器再次访问时，可以知己从本地加载，不需要再去服务端请求。若处理不当，会导致服务端代码更新，但用户还是老页面。

一、浏览器缓存的优点
  • 减少了冗余的数据传输，节省网费。
  • 减少了服务器的负担，提升网站性能。
  • 加快了客户端加载网页的速度。
    二、缓存位置
    四种，存在优先级，当依次查找缓存且都没有命中的时候才去请求网络。
    Service Worker：可以让我们自由控制缓存哪些文件、如何匹配缓存、如何读取缓存，并且缓存是持续性的。
    Memory Cache：内存缓存虽然读取高效，但缓存持续性很短，会随着进程的释放而释放。一旦关闭了Tab页面，内存中的缓存也就被释放了。
    Disk Cache：读取速度慢，但是什么都能存储到磁盘中，比内存缓存胜在容量和存储时效性上。
    Push Cache：当以上三种缓存都没有命中时，才会被使用。且缓存时间很短暂，只会在会话（session）中存在，一旦会话救赎就被释放。
    网络请求：以上缓存都没有命中，发送请求获取资源。
    三、缓存策略：强缓存和协商缓存
    1、强缓存：可以通过设置两种HTTP Header来实现，Expires和Cache-Control。
    强缓存如果命中，返回state code为200，客户端不会给服务器发送请求，直接从本地获取资源。
    Expries是一个时间戳，本地时间小于expires设置的过期时间，就说明命中强缓存，但本地时间改了会出错。
    Cache-Control优先级更高，max-age是一个时间长度，例如3153600秒代表在3153600秒内有效。
    S-maxage主要用于代理服务器生效。
    public：浏览器和CDN都可以被缓存。
    private：默认值，仅被览器缓存。

强缓存也称为本地缓存，发送请求时向浏览器缓存查找请求结果，并根据该结果的缓存规则来决定是否使用该缓存结果的过程。
强缓存的情况主要分三种：
①浏览器缓存中不存在缓存请求结果和缓存标识，则浏览器直接向服务器发送请求。
②浏览器缓存中存在请求结果和缓存标识，但根据缓存规则结果已经失效，则使用协商缓存。
③浏览器缓存中存在请求结果和缓存标识，且根据缓存规则结果尚在有效时间内，则直接返回请求结果。
2、协商缓存（弱缓存）：如果缓存过期了，需要发起请求验证资源是否有更新。可以通过谁知两种HTTP Header来实现：Last-Modified和Tag。当两者同时存在，服务器会先检查Etag，再检查Last-Modified，最终决定返回200还是304.
Last-Modified：服务器对比时间戳和资源在服务器上最后修改的时间是否一致，无法感知文件本身变化。
在浏览器第一次请求某个URL时，服务器端返回状态是200，内容是请求的资源，同时会有一个Last-Modified的属性标记此文件在服务器端最后被修改的时间。
客户端第二次请求此URL时，根据HTTP协议的规定，浏览器会向服务器传送If-Modified-Since报头，询问该时间之后文件是否有被修改过。
若服务器端的资源没有发生变化，则自动返回304状态码，内容为空，节省了传输数据量。
Etag：是资源的唯一表示字符串。

浏览器缓存中存在请求结果和缓存标识，但根据缓存规则结果已经失效，只返回缓存标识。
①浏览器携带该资源的缓存标识发起HTTP请求，服务器验证缓存标识发现资源无更新，返回304，不更新资源。
②浏览器携带该资源的缓存标识发起HTTP请求，服务器验证缓存标识发现资源更新了，则返回200，重新返回请求结果和缓存标识。

四、常用Git指令

git add 将工作区文件加入暂存区；
git commit 将暂存区文件加入本地仓库；
git push 将本地仓库推送到远程仓库。
git fetch 将远程仓库的更新下拉到本地仓库中远程副本，但不合并到本地分支当中，并更新FETCH_HEAD
本地搭建：
初始化一个Git仓库，使用git init命令。在一个文件夹内。使用UTF-8编码编辑文本，避免冲突（不用记事本）
添加文件到Git仓库，分两步
git add 添加多个文件时使用空格分开，也可以输入文件夹名称；
使用命令git commit -m ，上传本地仓库。
git status 可以查看本地仓库的状态，而如果有改动发生，则使用git diff这个命令
克隆远程仓库
在一个文件夹中打开git Bash并运行git clone [url]将克隆远程仓库并相关联
关联远程库
git remote add origin git@……可以关联到远程仓库， 与origin远程库关联之后，需要pull git pull origin 一下建立链接消除冲突， 然后将自己的代码push上去。git push –set-upstream origin
取消关联
git remote rm origin 即可取消与远程仓库的关联。
历史记录 git log
git log 查看历史记录
git log --pretty=oneline 参数得到一行信息 --graph获得拓扑图
--oneline 选项来查看历史记录的简洁的版本
git reflog 用来记录版本操作命令
git blame <file> 显示文件修改记录
工作区和暂存区
在电脑内部的文件目录就是工作，其中有一个版本库（本地仓库）存放了很多东西，其中最重要的就是称为stage（或index）的暂存区，还有Git为我们自动创建的第一个分支master，以及指向master的一个指针叫HEAD
我们使用 git add 就是将文件添加到暂存区，然后用 git commit 将暂存区的文件提交到分支上 （本地仓库）
